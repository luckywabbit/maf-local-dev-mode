OIDC Configuration

Set up OAuth/OIDC authentication for remote MAS environments

Overview

When connecting to a remote Maximo Application Suite (MAS) environment, you need to configure OAuth/OIDC authentication. This guide covers two approaches: automatic setup using OpenShift credentials, and manual configuration.

ℹ️
When do you need OIDC?
  • Local Manage: No OIDC setup required
  • Remote MAS: OIDC configuration is required

Prerequisites

🟢

Node.js

Version 20.0.0 or higher

Required for running the OIDC registration script.

🔑

MAS Credentials

Client ID & Secret

OAuth client credentials from your MAS administrator.

☸️

OpenShift CLI (Optional)

For automatic setup

Required only for automatic credential retrieval from OpenShift.

Automatic Setup with OpenShift

If you have access to the OpenShift cluster where MAS is deployed, you can automatically retrieve credentials from the cluster secrets.

Step 1: Install OpenShift CLI

If you don't have the OpenShift CLI installed:

# macOS (Homebrew)
brew install openshift-cli

# Linux
curl -LO https://mirror.openshift.com/pub/openshift-v4/clients/ocp/latest/openshift-client-linux.tar.gz
tar xvf openshift-client-linux.tar.gz
sudo mv oc /usr/local/bin/

# Windows (Chocolatey)
choco install openshift-cli

Step 2: Login to OpenShift

Authenticate with your OpenShift cluster:

oc login https://your-openshift-cluster.com:6443

You can get the login command from the OpenShift web console: Click your username → "Copy login command"

Step 3: Run Automatic Setup

Execute the OIDC setup script with automatic mode:

curl -fsSL https://ibm-mas.github.io/maf-local-dev-mode/scripts/oidc.sh | bash -s -- --auto

The script will automatically detect your MAS namespace and retrieve credentials from the cluster secrets.

✓
What gets retrieved automatically:
  • OAuth Client ID
  • OAuth Client Secret
  • MAS URL
  • Token endpoint URL

Manual Setup

If you don't have OpenShift access or prefer manual configuration, you can enter credentials directly. You'll need to obtain these from your MAS administrator.

Step 1: Obtain Credentials

Request the following information from your MAS administrator:

MAS URL
https://your-mas-instance.example.com
Client ID
maximo-dev-client
Client Secret
••••••••••••••••
Token URL (Optional)
https://your-mas-instance.example.com/oauth2/token

Step 2: Download Setup Script

Download the OIDC configuration script:

curl -fsSL https://ibm-mas.github.io/maf-local-dev-mode/scripts/oidc.sh -o oidc.sh
chmod +x oidc.sh

Step 3: Run Interactive Setup

Execute the script and choose manual entry:

./oidc.sh

The script will prompt you for each credential. The token URL will be auto-detected if you don't provide it.

Step 4: Verify Configuration

Check that your configuration was saved correctly:

cat ~/.maximo-dev-tools/oidc-config.json

You should see your MAS URL and client ID (the secret is encrypted).

Advanced Configuration

🔧 Multiple Environments

You can configure multiple MAS environments and switch between them. Each environment gets its own configuration profile. 

# Set up development environment
export MAS_ENV=dev
./oidc.sh

# Set up production environment
export MAS_ENV=prod
./oidc.sh

# Switch environments in VSCode
# Use Command Palette: "Maximo: Switch Environment"
🔒 Self-Signed Certificates

For development environments using self-signed certificates, you can configure the extension to trust them. 

# Add to ~/.maximo-dev-tools/oidc-config.json
{
  "masUrl": "https://your-mas.example.com",
  "clientId": "your-client-id",
  "rejectUnauthorized": false
}
⚠️
Security Warning:

Only disable certificate validation for trusted development environments.

⏱️ Token Refresh Settings

Customize token refresh behavior and expiration handling. 

# Add to ~/.maximo-dev-tools/oidc-config.json
{
  "tokenRefresh": {
    "autoRefresh": true,
    "refreshBeforeExpiry": 300,  // seconds
    "maxRetries": 3
  }
}
🌐 Proxy Configuration

Configure proxy settings for corporate networks. 

# Set environment variables
export HTTP_PROXY=http://proxy.example.com:8080
export HTTPS_PROXY=http://proxy.example.com:8080
export NO_PROXY=localhost,127.0.0.1

# Or add to VSCode settings.json
{
  "http.proxy": "http://proxy.example.com:8080",
  "http.proxyStrictSSL": false
}

Troubleshooting

❓ Authentication fails with "Invalid client credentials"

This usually means the Client ID or Client Secret is incorrect. Try:

  1. Verify credentials with your MAS administrator
  2. Check for extra spaces or hidden characters
  3. Re-run the setup: ./oidc.sh
  4. Check the config file: cat ~/.maximo-dev-tools/oidc-config.json
❓ Cannot connect to MAS - SSL certificate error

For self-signed certificates in development environments:

  1. Add "rejectUnauthorized": false to your config
  2. Or install the CA certificate in your system trust store
  3. Restart VSCode after making changes
❓ Token expires too quickly

The extension should automatically refresh tokens. If not:

  1. Check token refresh settings in config
  2. Manually refresh: Command Palette → "Maximo: Refresh Token"
  3. Check extension logs for errors
  4. Contact your MAS admin about token expiration policies
❓ OpenShift automatic setup fails

If automatic credential retrieval fails:

  1. Verify you're logged in: oc whoami
  2. Check you have access to the MAS namespace
  3. Try specifying namespace: ./get-credentials.sh mas-inst1-core
  4. Fall back to manual setup if needed
❓ Configuration file not found

If the extension can't find your configuration:

  1. Check file exists: ls -la ~/.maximo-dev-tools/
  2. Verify permissions: chmod 600 ~/.maximo-dev-tools/oidc-config.json
  3. Re-run setup if file is missing
  4. Check VSCode settings for custom config path
💡
Still having issues?

Check the extension logs: View → Output → "Maximo Dev Tools"
Or create an issue: GitHub Issues
